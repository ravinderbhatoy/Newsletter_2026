### Instructions to be Followed for Submitting Data for Newsletter 2026 (Vol. 13)

The Newsletter 2026 will include activities and achievements from **July 2025 to June 2026**.

To facilitate compilation, all departmental editors are requested to follow the instructions below.

### 1. Create a GitHub Account
Log in to your GitHub account OR Create an account at:  
[https://github.com/](https://github.com/)

### 2. Fork the Newsletter Repository
After logging into GitHub, visit the Newsletter repository:

[https://github.com/gndec-2021/Newsletter_2025](https://github.com/gndec-2021/Newsletter_2025)

Click the **Fork** button at the top-right corner.  
This will create a copy of the repository in your GitHub account.

### 3. Your Forked Repository
Your forked repository will appear as:

`https://github.com/YourUserID/Newsletter_2025`

Example:  
If GitHub username is **XYZ**, the link becomes:  
`https://github.com/XYZ/Newsletter_2025`

### 4. Working on the Repository
You may update content in either of the following ways:

**Option A – Browser Editing (Recommended for most users)**  
Work directly in GitHub using the online editor.

**Option B – Clone Repository to Local System (Optional)**  
Advanced users may clone the repository and work locally using a text editor.

To clone:
1. Click the green **Code** button in your forked repository.
2. Copy the URL under **Clone**.

Options available:
- HTTPS (recommended):  
  `https://github.com/YourUserID/Newsletter_2025.git`

- SSH (only if SSH keys are configured):  
  `git@github.com:YourUserID/Newsletter_2025.git`

### 5. Repository Structure
Inside the repository, open the folder named:

`Content`  

Within it, locate your department folder, such as:  

Example: Civil Engineering editor must work **only** inside folder `CE`.

⚠ **Do NOT modify or delete any other folder or file.**

### 6. Department Folder Contents
Each department folder contains:

- An `Images` folder
- A markdown file named after department abbreviation  
  (e.g., `CE.md` for Civil Engineering)

Use:
- `Images` folder for storing photographs
- Subfolders may be created (Events, Faculty, Labs, etc.) if required.

### 7. Editing Newsletter Content
Existing markdown files contain Newsletter 2024 content which may be used as reference.

Replace or update content for Newsletter 2025 accordingly.

Markdown syntax reference:  
https://www.markdownguide.org/cheat-sheet/

### 8. Editing Files in GitHub
To edit files in the browser:

1. Open the markdown file.
2. Click the **pencil icon (✏️)** to edit.
3. Make required changes.
4. Use **Preview changes** tab to verify formatting.

### 9. Commit Changes
After editing:

1. Write an appropriate **Commit message**  
   (Preferably include department name).
2. Select **Commit directly to the main branch**.
3. Click **Commit changes**.

Changes are automatically saved to your forked repository.

### 10. Review Your Changes
Verify formatting and content in your repository before submission.

### 11. Submit Updates via Pull Request
To submit your updates:

1. Go to your forked repository.
2. Click **Pull requests** tab.
3. Click **New pull request**.
4. Ensure:
   - Base repository: `gndec-2021/Newsletter_2025`
   - Head repository: Your fork
5. Click **Create pull request**.
6. Provide title and description if needed  
   (e.g., *Updated Civil Engineering content for Newsletter 2025*).

### Important Notes
- Edit **only your department folder**.
- Maintain formatting consistency.
- Ensure image names and file paths are correct.
- Preview content before submission.
